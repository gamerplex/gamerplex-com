SKILL.md — Build a skill game

The sovereign dev harness gives you a localhost:42069 chat + game-preview console wired to a local Anchor stack. You ship a game in 4 steps.

1. Clone the template

# public template under github.com/gamerplex
git clone https://github.com/gamerplex/gamerplex-dev
cd gamerplex-dev/templates/arcade-onchain
make dev # launches localnet + harness on :42069

2. Implement the GameState account

// programs/game/src/lib.rs
#[account]
pub struct GameState {
    pub p1: Pubkey,
    pub p2: Pubkey,
    pub status: u8,        // 0=pending 1=playing 2=finished
    pub winner: u8,        // 1=p1 2=p2
    pub score_p1: u32,
    pub score_p2: u32,
}

3. Register the adapter (one-time)

await cmProgram.methods
  .registerGameProgramV2({
    p1Offset: 8, p2Offset: 40,
    statusOffset: 72, statusFinishedValue: 2,
    winnerOffset: 73,
  })
  .rpc();

4. Anyone can settle

Once your game writes status = 2, anyone calls resolve_market_from_game_pda on CM v2.1 — payout direction comes from the on-chain winner byte, not the keeper. Trustless, permissionless, free.

📐 Design system rules — applied above

  • →One primary gradient CTA per screen — homepage ▶ Play Now, crash 💾 SAVE SCORE, lobby ⚔ START MATCH, market ⚡ BACK.
  • →Glassy purple cards at three intensities (subtle / standard / featured) per GLASSY_PURPLE_TRON.md. No new colors introduced.
  • →Tron grid mesh background everywhere, fades via radial mask. 48px on desktop, 32px on mobile.
  • →Per-game accent colors stay consistent: snake = green, chess = purple, words = yellow, locked = blurred-gold-hatch.
  • →Network badge always visible top-right. DEVNET orange now, MAINNET cyan when live.
  • →Skill-contest language only: "compete", "skill contest", "match", "prize pool", "service fee". No bet/wager/casino/predict copy on player surface.
  • →Mobile mockup at /_mockup-mobile.html shows the same components at phone breakpoints. Responsive bridge: 4-col coverflow → 2x2 grid below 900px.
